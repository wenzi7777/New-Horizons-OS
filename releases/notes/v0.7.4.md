# v0.7.4

## Changes

### Reliable Gateway Transfer

- Apply the `ttl_ms` deadline for `findme_switch_gateway` instead of keeping the
  preferred Gateway forever.
- Accept transfer offers only when both the target Gateway ID and claim ID
  match.
- Ignore expected rejection offers from non-target Gateways during handover.
- Clear the preferred Gateway and claim after timeout, then resume unbiased
  FindMe while retaining the previous Gateway host as a recovery target.
- Treat repeated commands for the same active or attached claim as idempotent.
- Add `preferred_gateway_id`, `transfer_active`, and
  `transfer_remaining_ms` to FindMe status.
