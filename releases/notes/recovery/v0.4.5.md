# New Horizons Recovery v0.4.5

- Stream OS write progress as compact `update_progress` messages and flush them while `write_os` is still running.
- Prevent the WebUI from marking the device offline during OS writes when the TCP control session is still alive.
- Report OS write failures as structured error `update_state` so the UI can show the failed file/state and allow a retry without a power cycle.

Requires New Horizons Runtime v0.3.0.
