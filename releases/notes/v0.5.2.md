# New Horizons OS Arduino v0.5.2

## Fixes

- Fixed a boot-window action button provisioning crash on ESP32-S3.
- The force setup portal path no longer pre-starts STA mode before SoftAP provisioning.
- The setup portal no longer calls `softAPdisconnect(true)` before an AP has actually been started.
- Device UID/AP SSID MAC reads now use `esp_read_mac()` so they do not depend on Wi-Fi stack state.

## Verification

- `python3 -m unittest discover -s tests`
- `arduino-cli compile --fqbn esp32:esp32:esp32s3:FlashSize=8M,PartitionScheme=default_8MB firmware/newhorizons_os`
