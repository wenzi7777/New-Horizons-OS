# New Horizons OS v0.4.13

- Treat STA link without DHCP IP as `wifi_link_no_ip` instead of resetting the interface during boot.
- Continue normal OS hardware startup when Wi-Fi is recovering, so matrix scan, IMU, battery, LED, logging, and local services can initialize before Gateway is online.
- Start FindMe, TCP, UDP, and NTP only after a valid STA IP is available, then attach automatically.
- Add Wi-Fi diagnostics to status, memory status, and scan health responses.
- Reject expired or wrong-mode boot commands so stale WebUI/Gateway requests cannot send a fresh OS boot back to Recovery.
