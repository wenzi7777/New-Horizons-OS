# New Horizons OS v0.2.28

- Replace device-side MQTT with UDP data streaming and a device-initiated TCP JSONL control session.
- Keep the existing binary sensor packet and server CSV schema unchanged.
- Send raw matrix packets over UDP so failed sends drop instead of blocking scan.
- Move command, status, result, update progress, files, and maintenance actions to TCP control.
- Configure New Horizons server host, TCP control port, and UDP data port from the Wi-Fi setup portal and WebUI server settings.
- Publish OS `.mpy` release files with `tcp_control.mpy` and `udp_stream.mpy`, and delete old `mqtt_transport` / `umqtt` files during update.
- Default stream ring buffer depth to 2 frames with native hard limit 8 for VD-CTL/R v1.0.F 2026.4.

Requires New Horizons Runtime v0.2.28 for the UDP/TCP transport and fixed native scan buffer changes.
