# New Horizons OS v0.4.4

- Add lightweight `scan_health` for max-matrix stress monitoring without polling full `status`.
- Report requested scan target, measured scan FPS, measured UDP FPS, produced frames, consumed frames, and dropped frames explicitly.
- Keep max-matrix scanning unconstrained by the OS profile while preserving IMU, battery, LED, logging, and HMAC.
- Improve UDP backpressure handling by dropping stale realtime stream frames and throttling transient send failure logs.
- Reduce allocation pressure in status and LED indicator paths during high-rate scanning.

Requires New Horizons Runtime v0.3.0. Recovery remains compatible with v0.4.3.
