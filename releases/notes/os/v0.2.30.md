# New Horizons OS v0.2.30

- Add emergency offline recording backed by a minimal `fs_core` layer.
- Store offline records as raw New Horizons binary packets in `data/offline`, not device-side CSV.
- Add offline recorder segment rotation, quota handling, oldest-segment rollover, drop counters, and status telemetry.
- Add SK6812 offline recording storage buckets for remaining-time feedback.
- Expose Offline Records in Device Files with raw segment download and server/WebUI-side CSV conversion.
- Keep full `filesystem_api` and `storage` confined to maintenance/files workflows; normal/offline paths use `fs_core`.

Requires New Horizons Runtime v0.2.28 or newer. Recovery remains compatible with v0.2.29.
