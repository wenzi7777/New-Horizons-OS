# New Horizons OS v0.4.2

- Fix TCP control sends on ESP32 nonblocking sockets by retrying `EAGAIN` / would-block writes instead of closing the FindMe attachment.
- Support partial TCP writes for hello, status, command result, and update progress JSONL messages.
- Avoid repeated FindMe reconnect churn after normal boot when the Gateway is reachable but lwIP is not immediately writable.
- Keep Runtime at v0.3.0; this update changes OS and Recovery payloads.

Requires New Horizons Runtime v0.3.0. Recovery v0.4.2 is recommended so recovery-mode OTA uses the same TCP control fix.
