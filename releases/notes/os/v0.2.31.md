# New Horizons OS v0.2.31

- Keep maintenance idle lighter: entering maintenance mode now stops scan and switches mode without loading filesystem or calibration services.
- Lazy-load filesystem only for file commands and calibration store only for calibration commands.
- Add calibration scan memory preflight so large matrix calibration returns a clear low-memory error instead of trying to start unsafe scans.
- Reduce `calibration_sample_all` per-frame Python allocation by reading native frame payloads directly.
- Keep status and maintenance status lightweight so diagnostics can run without pulling in heavy maintenance modules.

Requires New Horizons Runtime v0.2.28 or newer. Recovery remains compatible with v0.2.29.
