# New Horizons OS v0.2.27

- Stage matrix layout changes and only save them after the native scan core passes a health probe.
- Skip boot-time scan autostart for legacy or uncommitted matrix layouts, so a board with a bad saved 10x7/10x21 layout returns online instead of freezing again after hard reset.
- Disable scan autostart after an interrupted scan session so a hard reset returns the device online instead of boot-looping into a bad layout.
- Remove the normal-mode Python scan fallback and require the native stream core for matrix streaming.
- Publish raw sensor packets through the low-allocation QoS0 path while keeping command, status, and result messages on QoS1.
- Report scan memory and health information when matrix layout apply fails.

Requires New Horizons Runtime v0.2.27 because this release depends on the new native `pop_packet_into()` scan API.
