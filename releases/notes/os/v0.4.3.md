# New Horizons OS v0.4.3

- Stabilize the normal-mode TCP control channel with an outbound JSONL queue so temporary `EAGAIN` writes no longer close the FindMe attachment.
- Coalesce low-priority status announcements while preserving command results in the control queue.
- Add lightweight `memory_status` for RAM diagnostics so the WebUI no longer polls the full `status` payload every 3 seconds.
- Reduce normal-idle allocation pressure by slimming status announcements, lazy-loading storage from Wi-Fi portal helpers, and avoiding unnecessary battery refreshes.
- Add memory-stage logs around Wi-Fi, FindMe, LED, battery, IMU, and status setup to identify remaining idle RAM pressure.

Requires New Horizons Runtime v0.3.0. Recovery v0.4.3 is recommended so recovery-mode OTA uses the same TCP control queue behavior.
