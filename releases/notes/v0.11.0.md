# v0.11.0

## Features

### External LED indicator presets redesigned

The external WS2812B indicator presets were reworked to be honest, genuinely
dynamic, and to make use of all three LEDs. New preset list:
`system_status`, `connectivity`, `pressure_meter`, `stream_heartbeat`,
`calibration_auto`, `solid_marker`, `identify`, `off`.

- `system_status` (new default): one colour summarising overall device health,
  mapped from the already-prioritized system signal (connecting / no gateway /
  online / charging / error). Streaming warnings now use a recent-activity
  delta instead of a cumulative counter, so the warning state recovers on its
  own instead of sticking until reboot.
- `connectivity`: three-segment pipeline — LED 1 WiFi, LED 2 gateway, LED 3
  streaming (green / amber / red), so you can see at a glance where the link is
  stuck.
- `pressure_meter`: lights 1–3 LEDs from green to red based on the current
  matrix peak pressure — a real, live pressure indicator (replaces the static
  `pressure_activity`).
- `stream_heartbeat`: a teal heartbeat pulse while data is actively being sent.
- `calibration_auto`: amber breathing while a calibration session is active,
  off otherwise (replaces the manual `calibration_focus`).
- `solid_marker`: an honest fixed-colour marker with a selectable colour
  (teal / green / blue / purple / amber / red / white) for physically
  distinguishing devices (replaces `recording_focus`, which the firmware could
  never actually implement since it has no notion of "recording").

Implementation notes:
- New per-pixel render primitives `showSegments` / `showMeter` in
  `ExternalLedController`.
- `MatrixScanner` now exposes `lastPeak01()` (normalized peak pressure of the
  most recent frame), computed during the scan and reset when scanning stops.
- `service()` takes an `ExternalLedInputs` struct (system signal, WiFi/gateway
  state, pressure, calibration-session flag).
- A new `color` field was added to the external-LED config and protocol.

### Backward compatibility

Old preset strings stored on existing devices are aliased on load
(`stream_health` → `system_status`, `pressure_activity` → `pressure_meter`,
`calibration_focus` → `calibration_auto`, `recording_focus` → `solid_marker`)
via `DeviceConfig::canonicalExternalLedPreset`, so devices keep working after
the update and the desktop app rewrites the canonical string on next save.
