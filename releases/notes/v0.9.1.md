# v0.9.1

## Fixes

### Calibration persistence

- Shortened calibration storage paths under `/cal` so level profile atomic
  writes stay within the SPIFFS path limit.
- Preserved backward-compatible reads from the legacy `/calibration` profile
  layout and migrate the profile after the next successful commit.
- Changed calibration persistence so the legacy profile is cleaned only after
  the new profile has been written successfully, preventing failed writes from
  leaving an empty calibration profile.
- `sessionCommit()` now reports `calibration_save_failed` when persistence
  fails instead of treating the in-memory session as a durable commit.

### Storage diagnostics

- Limited automatic SPIFFS formatting to first-time initialization instead of
  silently formatting on every mount failure.
- Added `mounted`, `formatted_on_boot`, and `mount_error` to `storage_status`
  so storage health can be diagnosed from the control surface.
