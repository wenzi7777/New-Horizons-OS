# New Horizons OS Arduino v0.5.6

## Changes

- Changed fresh/default device configuration to leave the matrix layout unconfigured. A board with no saved pin layout now boots with an empty matrix and defers scanning instead of falling back to the full 10x21 VD-CTL/R layout.
- Added `matrix_layout.configured` to the persisted device config schema and migrated defaults to schema version 2. Older default-style config files without the explicit flag are treated as unconfigured.
- Updated scanner startup so `MatrixScanner` begins with zero rows and zero columns, only starts scanning when a real layout exists, and logs `scan_task_deferred matrix_layout_empty` when no layout has been applied.
- Improved FindMe discovery reliability by sending `findme_discover` to limited broadcast, directed subnet broadcast, and the previously accepted gateway host when known.
- Bumped firmware and release tooling defaults to `v0.5.6`.
