# v0.7.6

## Changes

### GCU v2.3.D LTS Flash Size Correction

- Correct the GCU v2.3.D LTS build FQBN from 8 MB to 4 MB
  (`FlashSize=4M,PartitionScheme=min_spiffs`) to match the actual hardware.
- The previous release (`v0.7.5`) used the stock 8 MB dual-OTA partition layout,
  which caused OTA updates to fail on 4 MB hardware after the first flash.
- No source-level logic changes. V1.0.F and V2.1 GCU LTS rebuilds are
  included to keep version strings consistent across all board tracks.

### GCU v2.1 LTS

- First versioned release for `VD-CTL/R v2.1 GCU LTS` on this track.
- 10 × 12 matrix, 4 MB Flash (`min_spiffs`), BMI270 + BMM150, no BQ25180.

## Verification

- Arduino scaffold and manifest tests pass.
- ESP32-S3 Arduino compile with 4 MB `min_spiffs` partition layout for:
  - `VD-CTL/R v2.3.D GCU LTS`
  - `VD-CTL/R v2.1 GCU LTS`
- ESP32-S3 Arduino compile with 8 MB `default_8MB` partition layout for:
  - `VD-CTL/R v1.0.F 2026.4`
