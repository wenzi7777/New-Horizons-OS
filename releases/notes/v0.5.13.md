# New Horizons OS Arduino v0.5.13

## Changes

- Fixed the WebUI scan timing apply flow so a freshly applied `target_fps` is not immediately overwritten by an older device snapshot.
- Changed BMI270 sampling from per-frame inline reads to a continuous-mode cached service cadence, reducing scan-loop blocking when IMU is enabled.
- Added IMU runtime diagnostics for sample rate, cache age, last read duration, and last error so performance regressions can be verified from status.

## Verification

- `python -m unittest discover -s apps/newhorizons/tests`
- `python -m unittest discover -s NewHorizonsOS-OTA/tests`
- `npm run build`
- `arduino-cli compile --fqbn esp32:esp32:esp32s3:FlashSize=8M,PartitionScheme=default_8MB firmware/newhorizons_os`
