# New Horizons OS v0.6.5

## Changes

- Fixed large UDP command results being lost. Big command responses (notably `status`, ~2.5 KB) exceed a single UDP datagram and were dropped to IP fragmentation over WiFi/Docker-bridge paths, leaving the desktop UI (Pin Layout, Diagnostics) blank. The control server now splits any oversized result into application-level `result_chunk` datagrams (each well under the MTU, keyed by `request_id`); the gateway reassembles them before forwarding. Small results are unchanged and still sent inline. Requires Gateway v0.1.9+ for reassembly.
