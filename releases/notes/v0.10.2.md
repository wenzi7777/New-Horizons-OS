# v0.10.2

## Fixes

### Calibration lost after a true power cycle

- Fixed calibration being dropped on cold boot. `Calibration::loadFromStorage`
  enumerated the `/cal` directory with `File::name()`, which on the current
  Arduino-ESP32 core (ESP-IDF v5.5.1) returns only the basename, so the tare and
  level files never matched the expected full paths (`/cal/tare`, `/cal/l*`) and
  were never read back. The metadata loaded, but with empty data
  `complete()` was false and calibration was force-disabled.
- Switched the directory enumeration to `File::path()` (full path) in the load
  path, matching the already-correct usage in `Storage::directorySize`.
- Applied the same `File::name()` → `File::path()` fix to
  `removeStoredLevels` and `removeObsoleteStoredFiles`, whose `SPIFFS.remove()`
  and keep checks were silently failing and leaving obsolete calibration files
  behind.
- Added a `cal_loaded ...` serial summary line after load so cold-boot
  calibration recovery can be confirmed over the serial console.

The previous behaviour only appeared to persist across a "soft off / soft on"
cycle because soft-off uses light sleep (RAM is retained and `setup()` never
re-runs), which masked the broken durable read. Any real power cut (power
switch / battery removal) exposed the loss. Calibration data already written to
flash is intact, so devices recover their existing calibration after updating —
no re-calibration required.
