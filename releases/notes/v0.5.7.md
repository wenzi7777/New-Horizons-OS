# New Horizons OS Arduino v0.5.7

## Changes

- Kept the stock ESP32-S3 8MB `default_8MB` partition scheme: dual OTA apps with the existing 3MB app slots and 1.5MB SPIFFS.
- Added persistent rolling device log settings: enabled by default, standard 16KB mode, extended 32KB mode, and configurable log level.
- Added persistent OTA settings and optional boot-time auto update check/apply using the Arduino whole-firmware manifest.
- Fixed fresh devices with no saved matrix layout: applying a matrix layout now starts scanning immediately without requiring a reboot.
- Expanded WebUI maintenance controls for calibration, logs, storage status, and file browser access.
- Removed the unused IMU filter control from WebUI/backend command surfaces.
- Reduced large-matrix visualization churn by coalescing 10x21 frontend frame commits, lowering surface interpolation density for large matrices, and reusing Three.js geometry buffers.
