# New Horizons OS Arduino v0.5.10

## Changes

- Fixed manual `apply_update` over the Arduino TCP control path so the board acknowledges `update_started` immediately, then performs OTA after the control socket closes instead of blocking until timeout.
- Fixed boot-time auto OTA so the firmware now enforces download progress timeouts and reports `firmware_download_timeout` or `firmware_download_overall_timeout` instead of hanging indefinitely on a stalled HTTP stream.
- Added OTA state and observability improvements for both paths, including `operation/current_file/last_result/reboot_required` status fields plus internal LED signaling for auto OTA active, success, and error states.
- Updated the WebUI backend OTA state merge so `update_started` is treated as in-progress and a reconnect on the target firmware version is normalized to `apply_update -> done`.
