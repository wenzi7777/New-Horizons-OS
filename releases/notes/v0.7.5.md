# v0.7.5

## Changes

### GCU v2.3.D LTS Board Support

- Add a board capability profile so the Arduino firmware can target both
  `VD-CTL/R v1.0.F 2026.4` and `VD-CTL/R v2.3.D GCU LTS` from the same runtime.
- Add default `15 x 15` matrix layout and pin mapping for GCU LTS while keeping
  the existing `10 x 21` layout for v1.0.F.
- Keep the packet v3 wire contract stable: IMU stays a 7-float base payload and
  GCU boards may append a 3-float magnetometer block behind `kPacketFlagMag`.
- Add buttonless boot/provisioning and soft-off handling for boards without the
  legacy local action button.
- Make OLED and external LED components compile-time safe no-ops on capability-off
  boards instead of keeping hard dependencies in the main build path.
- Add a dedicated GCU LTS release build and manifest track so OTA model checks no
  longer fail with `model_mismatch`.

## Verification

- Arduino scaffold and manifest tests.
- Desktop Arduino packet v3 parser tests.
- ESP32-S3 Arduino compile with the stock 8 MB dual-OTA partition layout for:
  - `VD-CTL/R v1.0.F 2026.4`
  - `VD-CTL/R v2.3.D GCU LTS`
