# v0.6.11

## Features

### Power Transition Feedback
- Added a short shutdown animation for soft-off transitions. The board `SK6812B`, external `WS2812B`, and OLED now each play a capability-aware power-off sequence before the device enters light sleep.
- Added a wake animation specifically for `soft-off -> short press -> wake`, so pressing the action button now gives immediate visible feedback before the normal runtime indicators take over.
- Added structured Serial power lifecycle logs covering button down/up timing, requested transitions, animation start/end, runtime suspend/resume, and soft-off wake causes.

### Log Storage and WebUI
- Changed persistent log storage presets to `standard = 12KB` and `extended = 24KB`, while keeping the existing dual-file rotation model (`device.log` + `device.log.1`).
- Changed persistent log defaults to `enabled = true`, `mode = standard`, and `level = error`. Serial output still prints regardless of storage log level.
- Added `effective_total_bytes` to the device log status payload so the WebUI can show the real flash budget under dual-file rotation.
- Updated the Desktop WebUI to show `warn`/`error`/`info`/`debug`, display the real flash footprint for each log mode, and open log preview by default in the file browser `logs` scope with a manual close option.
