# New Horizons OS Arduino v0.5.9

## Changes

- Fixed `set_indicators` float parsing so numeric brightness values like `0.10`, `0.35`, and `1.00` are read from JSON correctly instead of falling through to the next quoted key and collapsing to `0.0`.
- Fixed external WS2812B brightness persistence and command handling for Arduino control payloads that send decimal brightness values before other indicator keys.
- Added regression coverage for the Arduino control-server float parser so future indicator payload changes keep parsing numeric brightness from the request body.
