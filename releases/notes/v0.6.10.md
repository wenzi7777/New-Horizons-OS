# v0.6.10

## Bug Fixes

### FPS Accuracy at 60fps Target
- Fixed 60fps scan rate reporting ~48fps actual. Root cause: `yield()` on every loop iteration allowed the WiFi FreeRTOS task (priority 23) to preempt the scan loop, accumulating ~4ms of delay per 16.67ms interval. Fix: only yield when more than 2ms remains until the next scan, ensuring the loop runs at full speed as scan time approaches.
- Added IMU service interval sync to scan interval (`imu.setServiceIntervalUs(scanner.scanIntervalUs())`), ensuring IMU I2C reads occur in the same loop iteration as the scan rather than between scans.
- Both fixes apply at boot and when scan timing is updated via `set_scan_timing` command.

### Soft-Off Button Wake Reliability
- Fixed short-press wake from soft-off requiring multiple presses. Root cause: after GPIO wakeup (button down), `lightSleep()` re-entered with the full 2-second battery sleep timer. The button release during that sleep was missed, making `heldMs` appear as ~2000ms which exceeded `kShortPressMaxMs` (500ms). Fix: use a 50ms poll timer when `buttonDown_` is true, so button release is always detected within the valid short-press window (50–500ms).
