# v0.8.0

## Changes

### Pressure Calibration: Auto-apply Range After Calibration Completes

- After completing calibration in the Desktop visualization page, the calibration
  range is now automatically applied without requiring manual adjustment.
- Implemented by issuing an automatic `calibration_status` query after session
  commit, so the frontend immediately receives the updated `enabled: true` state.

### Pressure Calibration: Persist Calibration Across Power Cycles

- Fixed a firmware initialization-order bug that caused calibration to appear
  incomplete after a power cycle, even though calibration data was correctly
  saved to SPIFFS.
- Root cause: `calibration.begin()` (which calls `loadFromStorage()`) was invoked
  before `calibration.setLayout()`, so `totalPointCount()` was zero during load,
  causing tare and level data to be silently discarded and `legacyMissingTare_`
  to be incorrectly set to `true`.
- Fix: swapped initialization order so `setLayout()` runs before `begin()`, and
  added a guard in `loadFromStorage()` to avoid overwriting already-set pin
  counts from the meta file.

### Pressure Calibration: Tare (Zero-point Reset) Function

- Added a new `calibration_tare_capture` command that captures the current
  sensor readings as the zero-pressure baseline (tare) and persists it to SPIFFS,
  without requiring an active calibration session.
- New "Apply tare" button added in the device maintenance interface (Settings →
  Maintenance). Requires maintenance mode. Samples sensors for 1 second and
  writes the result immediately.

## Verification

- Arduino scaffold and manifest tests pass.
- ESP32-S3 Arduino compile for all four board models.
