# New Horizons OS Arduino v0.6.4

## Changes

- Added serial logging (`control_command_received`, `control_command_executing`, `control_command_finished`) and LED feedback (`CommandReceived`, `CommandSuccess`/`CommandFailed`) to the UDP command path, matching the existing TCP path behaviour.
- Added `servicePendingApplyUpdate()` call in the UDP command handler so OTA triggered via UDP is applied correctly.

## Verification

- `python3 -m unittest discover -s tests -q`
- `arduino-cli compile --fqbn esp32:esp32:esp32s3:FlashSize=8M,PartitionScheme=default_8MB firmware/newhorizons_os`
- `VERSION=v0.6.4 firmware/scripts/build_arduino_release.sh`
- `python3 firmware/scripts/generate_arduino_manifest.py --firmware releases/artifacts/newhorizons-os-v0.6.4.bin --output releases/arduino-v0.6.4.json --version v0.6.4 --base-url https://raw.githubusercontent.com/wenzi7777/New-Horizons-OS/v0.6.4/releases/artifacts --changelog-url https://raw.githubusercontent.com/wenzi7777/New-Horizons-OS/v0.6.4/releases/notes/v0.6.4.md`
